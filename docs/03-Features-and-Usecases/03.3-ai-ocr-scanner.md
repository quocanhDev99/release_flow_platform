# AI OCR Scanner

## 1. Feature Overview
The AI OCR Scanner allows users to bypass manual entry entirely by uploading images (e.g., screenshots of Excel spreadsheets or emails). The system uses external AI Vision APIs to extract the deployment schedules and import them directly into the database.

## 2. Use Case Diagram

```mermaid
usecase
  actor "Release Manager" as RM
  actor "System" as SYS
  actor "Google Gemini/Vision API" as AI

  package "AI OCR Scanner" {
    usecase "Upload Image" as UC1
    usecase "Process Image via AI" as UC2
    usecase "Parse JSON Output" as UC3
    usecase "Preview Extracted Data" as UC4
    usecase "Confirm Import" as UC5
  }

  RM --> UC1
  RM --> UC4
  RM --> UC5

  SYS --> UC2
  SYS --> UC3

  UC1 ..> UC2 : <<include>>
  UC2 --> AI : Calls API
  UC3 ..> UC4 : <<include>>
```

## 3. Sequence Diagram (Extract and Import)

```mermaid
sequenceDiagram
    autonumber
    actor RM as Release Manager
    participant Frontend as Angular UI
    participant Backend as NestJS API
    participant AI as Gemini Vision API
    participant DB as PostgreSQL (Prisma)
    participant Notifications as Notification Service

    RM->>Frontend: Drag & Drop Spreadsheet Screenshot
    Frontend->>Backend: POST /api/ai/extract-schedule (FormData)
    
    Backend->>Backend: Validate Image Format & Size
    Backend->>AI: Send Image with extraction prompt
    AI-->>Backend: Return JSON structure of schedules
    
    Backend->>Backend: Map JSON to Internal Model
    Backend-->>Frontend: 200 OK (Extracted Data Array)
    
    Frontend->>Frontend: Display Data Preview Table
    RM->>Frontend: Review data, adjust if needed, click "Import"
    
    Frontend->>Backend: POST /api/schedules/bulk-import (JSON Array)
    Backend->>DB: INSERT multiple DeploymentWindow & Bookings
    DB-->>Backend: Success
    
    Backend->>Notifications: Trigger AI Import Notification (Broadcast)
    Notifications-->>Backend: Success
    
    Backend-->>Frontend: 201 Created
    Frontend-->>RM: Show Success Message & Update Dashboard
```
