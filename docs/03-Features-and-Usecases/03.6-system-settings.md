# System Settings UI

## 1. Feature Overview
The System Settings feature provides an administrative interface to configure global application variables, such as Webhook URLs (Slack, Teams), API Tokens (Telegram), and SMTP credentials. This allows the system to be dynamically configured at runtime without requiring `.env` file changes or application restarts.

## 3. Sequence Diagram (Update & Test Settings)

```mermaid
sequenceDiagram
    autonumber
    actor Admin
    participant Frontend as Angular UI
    participant SettingsAPI as Settings Controller
    participant DB as PostgreSQL (Prisma)
    participant NotificationSvc as Notification Service

    Admin->>Frontend: Navigate to "Settings > Alerts & Notifications"
    Frontend->>SettingsAPI: GET /api/settings
    SettingsAPI->>DB: Fetch global settings
    DB-->>SettingsAPI: Return settings object
    SettingsAPI-->>Frontend: 200 OK (Settings Data)
    
    Admin->>Frontend: Modify Slack Webhook URL & Click "Save"
    Frontend->>SettingsAPI: POST /api/settings (Payload)
    SettingsAPI->>DB: Upsert Configuration Keys
    DB-->>SettingsAPI: Success
    SettingsAPI-->>Frontend: 200 OK
    Frontend-->>Admin: Show Success Toast
    
    Admin->>Frontend: Click "Test Slack"
    Frontend->>SettingsAPI: POST /api/settings/test-slack
    SettingsAPI->>NotificationSvc: Send test payload to saved webhook
    NotificationSvc-->>SettingsAPI: HTTP 200 OK
    SettingsAPI-->>Frontend: 200 OK
    Frontend-->>Admin: Show Test Success Message
```
